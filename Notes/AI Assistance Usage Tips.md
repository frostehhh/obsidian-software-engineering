## Use AI to ask about your codebase
This is useful to familiarize with the capabilities of the AI assistant or model you are using

## Make a plan beforehand and ask for confirmation
Explore > Plan > Confirmation > implementation

# Context
## Specify folders, files, etc. if possible
- With provided context, performance will be better
## Include context rules on how to use your tools
- Include rules or context for how to use tools
- In Cursor, we can add this to https://docs.cursor.com/context/rules
## Define your project architecture and style
- Be as specific as needed as if you are designing an internal documentation
## Take time to optimize your context

# Testing
## Prompt AI to create test cases prior to creating a feature
With test cases, the AI will have a basis to refer to to indicate if the feature is working as expected or not.
