- System design is about deciding what we need and why we need them while considering tradeoffs

# Outline
1. Decide what are the system's constraints and use cases
2. Given the use cases, can use back-of-the-envelope estimates
3. Construct a high-level architecture
4. Details for core components
5. Load test
6. Incrementally scale

> [!tip]
> The steps are synonymous to identifying the problem, high-level overview and detailed implementation.
