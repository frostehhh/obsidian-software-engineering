- GraphQL allows for flexible querying of data
- Users can specify the structure of the data they need
- Useful for quickly iterating frontend - flexibility

- On the contrary, when using REST API, when frontend needs new data, we may need a new endpoint on the backend to represent the data.
	- There are also cases where we create multiple data requests
		Example: Request user -> 10 different requests for getting user details
	- GraphQL can simplify this into 1 request
