# Learning Tests
- Tests created to verify assumptions of a system's capabilities
- Great to be done during research phase of a workflow
- High-leverage by making verified conclusions in early stages of development
- Verifying early avoids nightmares such as rewriting more than 50% of your implementation due to one faulty assumption
- Results from learning tests can be documented as an `.md` file or a skill for future use
- Use when
	- you have no access to the source code of APIs, tools
	- you need to confirm a system's functionalities
