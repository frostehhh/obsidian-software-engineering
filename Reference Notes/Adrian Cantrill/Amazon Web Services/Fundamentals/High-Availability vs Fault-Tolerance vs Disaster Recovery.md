# High Availability
![[Pasted image 20230326184754.png]]
- Maximizing a system's online time
- Not related to user experience
- Minimizing outages
- Fast or automatic recovery of issues
- High availability systems can still have outages


# Fault Tolerance
![[Pasted image 20230326190147.png]]
- property that enables a system to continue operating propely in the event of the failure of some or more of its components
- Similar to [[High-Availability vs Fault-Tolerance vs Disaster Recovery#High Availability|High-Availability]] where we minimize outages

# Disaster Recovery
![[Pasted image 20230326190201.png]]
- Set of policies tools and procedures to enable the recovery or continuation of vital technology infrastructure and systems following a natural or human-induced disaster.
- Keep the crucial and non-replaceable components of a system safe
