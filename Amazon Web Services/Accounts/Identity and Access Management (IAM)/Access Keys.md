[[Identity and Access Management|IAM]] [[Access Keys]] - this is how authentication via command line

- long-term credentials
- An [[Identity and Access Management|IAM]] [[User]] can have two access keys
- Access keys can be created, deleted, made inactive or made active
- With a given Access Key ID, a secret access key is given as a pair. The secret access key is allowed to be downloaded only once.
- [[Identity and Access Management|IAM]] [[User]]s are the only identities that can use access keys. This includes an [[AWS Account]]s root user.
