Mockito is a mocking framework intended to be used with testing frameworks like JUnit and TestNG.

# Concepts
- Stub - an object that provides a predefined response for a method
- mock - an object that verifies behaviors. This may involve creation of stubs

# Core functionalities
## Creating and Injecting Mocks
`@InjectMocks` 
- injects mocks onto the object this annotates
- field-level annotation
`@Mock`
- creates a mock of the annotated object
- field-level annotation
`Mockito.mock(...)`

## Stubbing Values
With a mocked object:
`when(...).thenReturn()`

```java
@Mock
CampaignRepository campaignRepository;

@InjectmMocks
DefaultCampaignService campaignService;

@Test
void givenANewCampaign_whenSaving_thenSuccess() {
	// given
	//...
	
	when(campaignRepository.findById(3L)).thenReturn(newCampaign);
	// when
	// save and find
	
	// then
	assertEquals(newCampaign, foundCampaign)
}
```

## Stubbing voids
This is to explicitly mock that we expect nothing to be returned, but we still want the given method be invoked
`doNothing().when(...)`

## Verifying behavior
- Check if a certain method is invoked with specific arguments
- can use argument matchers
`verify(...)`

## Advanced Stubbing
Instead of `thenReturn`, use `thenAnswer`. Here, the arguments and the returned value can be modified. This is to simulate side-effects
If you need to modify void method behaviors, use `doAnswer` instead.
