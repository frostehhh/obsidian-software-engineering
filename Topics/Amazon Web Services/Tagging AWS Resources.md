Tags are key and value pairs that act as metadata for organizing your AWS resources. With most AWS resources, you have the option of adding tags when you create the resource.

# Best Practices
- Do not add personally identifiable information (PII)
- Use too many tags rather than too few tags
- You can use [[AWS Organizations]] to enforce tag policies you can either monitor tags according to configured standards or prevent non-compliant tags from being created.
See other best practices in the attached reference

# Tag naming limits and requirements
- 50 maximum user-created tags per resource
- Tag keys on a resource must be unique
- Allowed characters can vary by resource
- Tag key and values are case-sensitive
